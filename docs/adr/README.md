# Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) for TaskFlow.

ADRs document significant architectural decisions made during development, including the context, decision, and consequences.

| [ADR-0003](0003-undo-redo-system.md) | Undo/Redo System Design | Accepted |

## ADR Format

Each ADR follows this template:

```markdown
# ADR-XXXX: Title

## Status
Accepted | Proposed | Deprecated | Superseded

## Context
What is the issue that we're seeing that is motivating this decision?

## Decision
What is the change that we're proposing and/or doing?

## Consequences
What becomes easier or more difficult because of this change?
```
